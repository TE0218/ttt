How to use global procedures - 2

  • Local procedures can be converted to methods that can be used by global procedures.

    • To convert a local procedure use the button “Convert local procedure to method” in the “Mapping procedure” tag of the mapping.

    • Procedure call in the mapping needs to be changed!

    • Local procedure can be deleted after conversion to be a method.

  • This process also works in reverse:

    • Methods of global procedures can be converted to local procedures.

    • In that case the procedure call need to be changed as well!

For more details please refer to Global Procedures

Global procedures provide reusable mapping code. In detail, global procedures contain a set of methods and those methods carry the code.

The code can be reused by many mappings. If a global procedure is changed, only this global procedure has to be compiled. The mapping will contain those changes without it having to be recompiled.

The methods are written like the record program of a mapping. They can use the same functionality (commands/functions) and have access to the current source record and the destination structure of the mapping, which calls the method of the global procedure.

Note: They cannot use global or dim variables of the mapping where they are used!

Global procedures can be imported and exported from or to the file system. To share them with other mapping developers, they can be exported/imported to the repository.

If they should be used in the BIS 6 system, they have to be actived in the Mapping Designer and then deployed in the BIS 6 Front-end. This is described in the manual BIC Mapping Repository.
