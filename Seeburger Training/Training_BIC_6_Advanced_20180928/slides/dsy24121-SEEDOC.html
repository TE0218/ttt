CSV

The CSV message type contains comma- (or semicolon-)separated file message types as generated by Microsoft Excel. It also contains any supported message type with variable length that is not EDIFACT, ANSI X12, AECMA or TRADACOMS type.

CSV files consist of records, like all conventional sequential ASCII files. These records consist of single fields. Every record can start with a keyword , e.g. 100 (the record identifier). Every record (e. g. 100) is delimited by a special delimiter, e. g. apostrophe (').
This delimiter can also be Carriage Return Line Feed ("\r\n" segment delimiter).

Records consist of:

  • Data fields
  • Composite data fields

Record identifier, composite data fields, and data fields are delimited by special characters like the plus symbol ("+") (Element delimiter).
Composite data fields consist of sub-fields. Sub-fields are delimited by a further delimiter, e. g. colon (":") (SubElement delimiter).
Repeated composite data fields and data fields are delimited by special characters like asterisk (" * ") (Group delimiter).

Further information about the key rules is described in the chapter key rules of  Rules and Functions for Flatfiles (INHOUSE).

Conditional Fields

Although not defined in CSV, the belonging message formats allow a definition of field conditions as known from the ANSI X12 Conditional Fields norm. For details see chapter Conditional Fields.
