How to use local procedures - 2

  • Procedure will be called by write down the procedure name followed by brackets inside the mapping.

Example:
WRITE_ADDRESSDATA()

For more details please refer to Procedures

Procedures provide a possibility to write reusable code. There are two kinds of procedures within the BIC Mapping Designer: Global procedures and local procedures.

Global procedures can be used by all mappings, local procedures are placed in one mapping, and can only be used by this mapping.

Procedure Editor

The Procedure Editor is used to create and modify the procedure code.

For global procedures it is shown whenever a global procedure is opened (via the context menu, or with a double mouse-click from the Project Explorer). It contains a list view of all existing methods in that global procedure, a variables view, and the source code editor for the currently selected method. It can be seen as a small mapping view.

The local Procedure Editor is integrated into the mapping view in the tabs Procedure source and Mapping procedures. If one of these tabs is selected, both tabs are shown in the Mapping Editor. The Procedure source tab at the bottom of the mapping view contains the source code of the current selected procedure. The Mapping procedures tab at the right upper part of the mapping view contains the list of all existing local procedures.

Methods inside a global procedure are treated the same way as local procedures in a mapping. Therefore the Editor part of the Global procedure view has basically the same meaning as the Procedure source tab in a mapping view. Furthermore, the Mapping procedures tab in the mapping view has the same functionality as the Global procedure methods part in a global procedure view. 

The Mapping Procedures/Global Procedure Methods View

In this view the list of all available methods/local procedures are shown. It is possible to edit or delete an existing, or create a new method/local procedure, if the corresponding button is clicked.

Every Edit or New process opens the properties view of a method/local procedure. It is possible to change the method/procedure signature (that means name, description, return type and parameters) within that view. It is not allowed to rename a method/local procedure. Please delete that method/procedure, and create a new one instead.

Note: The BIC Mapping Designer ensures that newly created procedures are named uniquely. It is not allowed to use identical names for procedures, BIC commands and functions (refer to the topic Commands in the Mapping Designer Command Reference), or variables.

Note: Every signature change of a method or local procedure may not be backward compatible. You have to check every affected mapping or global procedure.

Possible return values types are:

  • Numerical
  • Alphanunmerical
  • BigDecimal
  • Boolean
  • void

Procedure parameters can be added, removed, or edited on the second tab of the Procedure Property panel. The characters "~", "$", "%", "?", "[","]", "{" and "}" are not allowed within the parameter name. They cannot be entered in the text field. The characters "$", "%", "~" ,"?", "[","]", "{" and "}" are automatically added to the parameter names, "%" means numeric type, "$" alphanumeric type,  "~" bigdecimal type, "?" boolean type, "[  ]" array and "{ }" hashmap, if both ("[ ]" and "{ }") are use, it is a mixed type of array(s) and hashmap(s).

The following parameter types can be applied:

  • Numerical
  • Alphanunmerical
  • BigDecimal
  • Boolean

Simple parameters (no) can be passed by value or by reference.
By value means they are not changed within the procedure/method. If a parameter is passed by reference, it can be changed within the procedure. Because this may lead to unexpected results and effects, it is recommended to add a corresponding comment to the description part of the method/procedure.

Arrays, hashmaps and mixed types are always passed by reference.

Attention: If a local procedure belongs to a mapping with programming language BIC-BASIC V1, the new dynamic arrays,hashmaps and mixed types are not supported. The simple parameters are always passed by value and the array parameters by reference.  If a wrong usage of the parameters is detected, an error dialog is shown, which contains information about the wrong usage.
