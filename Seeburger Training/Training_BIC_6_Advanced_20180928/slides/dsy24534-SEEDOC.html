How to use return values

  • Procedure execution (local and global ones) allows return values. The return value will be passed back to the mapping for further usage.
  • To use a return value it is necessary to use the "exitproc" command.
    If no return value is defined, the "exitproc" command is not needed.
  • The following return types are possible:
    • Alphanumeric string
    • Integer number
    • Boolean
    • BigDecimal
    • Void
  • SYNTAX:
    exitproc(returnValue);
    exitproc();
  • For using a return value with a global procedures, the return value needs to be picked up in the mapping explicitly.
    • SYNTAX:
      returnValue = [Global_Procedure_Name]_[Method_Name]( [Parameter]…);

To anticipate problems in exchanging information between mapping and procedure it is better to use input/return values instead of global variables.

For more details please refer to exitproc Syntax

exitproc()

exitproc(returnValue)

Description

Returns, at the end of the procedure, the return value of the procedure to the mapping. If no return value of the procedure is defined, the exitproc command is not needed, but it could be used (without a parameter).

The following return type are possible:

  • Alphanumeric string
  • Integer number
  • Boolean
  • BigDecimal
  • Void
Parameter(s)

Parameter

Type

Description

returnValue

$,%,?,~

The value, which is returned by the local procedure of a mapping, or by a method of a global procedure.
Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
local retValue$;
// method/procedure code
//...
exitproc(retValue$);

or

exitproc();

To exit a procedure early, the exitproc command can be used earlier in the code (with, or without a return value). The procedure is exited, and the mapping continues with the next command after the procedure call.

local kkk%;
//...
if kkk% = 1
	exitproc();
endif
