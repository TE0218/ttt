Error mappings - Exercise

  1. To test the error mapping handling you have to adjust the main mapping from the previous exercise to handle also the invoice list of the incoming file. Please delete the code line “stopreading();” for the case “393” at the BGM segment and insert an error command that this kind of messages are not supported.

  2. You also have to change the option stopreadingonerror to off to handle all errors which are thrown at the main mapping.

  3. As next step you have to import the error mapping from the class material (INVOIC_SERVICE_ERROR).

  4. To link the error mapping to the main mapping set the global option at the corresponding tab.

  5. Test your mappings with the file INVOIC_D01B.txt and check the logfile.
