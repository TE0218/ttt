getMTField(key)

Description

It is used to get a field/column from the current message tracking database record. The database has to have already been connected with the openMTDBConnection command before. If no record is present, a new record is created. If no value could be found for the given key, the returned value is a string, or an empty string. Please refer also to the topicFunctions for Message Tracking Databases.

Parameter(s)

Parameter

Type

Description

key

$

The name of the column to get.
Return Value

Type

Description

$

Value of the field/column defined by the key.
Warning(s) / Error(s)

If the field does not exist, the warning

  • 2601

is created. In this case an empty string is returned.

Example

dim Text$;
Text$ = getMTField("Info4");
clearMTRecord Syntax

clearMTRecord()

Description

Clears the fields of the current Message Tracking record. The database has to have been connected, via the openMTDBConnection command, previously. Please refer also to the topic Functions for Message Tracking Database, especially the command executeMTCommandWithMultiResult .

Parameter(s)

This command does not expect any parameters.

Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example: 
clearMTRecord();
closeMTDBConnection Syntax

closeMTDBConnection()

Description

This command is used to close a connection of a Message Tracking database (if it is still open due to the openMTDBConnection command). Refer also to the other Functions for Message Tracking Database.

Parameter(s)

This command does not expect any parameters.

Return Value

This command does not return a value.

Warning(s) /  Error(s)

If the closeMTDBConnection call returns an MTException the error

  • 2613

is created. The MTException is thrown, if the closing process fails, or the Message Tracking is working in an asynchronous mode.

Example 
closeMTDBConnection();
