DTD - Document Type Definition

A Document Type Definition (DTD) defines the legal building blocks of an XML document. It defines the document structure with a list of legal elements and attributes. A DTD can be declared inline inside an XML document, or as an external reference.

The following example is a DTD file called "PART2_DTD.dtd" that defines the elements of the XML document the part before:

<?xml encoding="UTF-8" ?>
<!ELEMENT root (customer,supplier?)>

<!ELEMENT customer (name,address*,volume)>
<!ATTLIST customer
	id 	CDATA 	#IMPLIED>

<!ELEMENT name (#PCDATA)>

<!ELEMENT address (#PCDATA)>
<!ATTLIST address
	str 	CDATA 	#IMPLIED>

<!ELEMENT volume (#PCDATA)>
<!ATTLIST volume
	value 	CDATA 	#IMPLIED>

<!ELEMENT supplier (name,address*,volume)>
<!ATTLIST supplier
	id 	CDATA 	#IMPLIED>

The first line defines the note element to have two child elements: "customer,supplier".
