writeMTAdditionalDetailRecord(primaryKey, masterKey)

Description

It is used to write the current Message Tracking's database record to the Message Tracking's additional detail database table (tBISMT_Documents2). The database has to be connected via the openMTDBConnection command before. If the record is already present, an update (otherwise an insert) is performed. Please refer also to the topicFunctions for Message Tracking Databases.

Parameter(s)

Parameter

Type

Description

primaryKey

$

The primary key of the database record, which should be written.

masterKey

$

The primary key of the master database record. Also null or "" is applicable.

Return Value

This command does not return a value.

Warning(s) / Error(s)

If the connection cannot be established, an SQL error occurs or no current record is present, the error

  • 2614

is created.

Example 
local primKey$;
local masterKey$;
//keys have to be set
writeMTAdditionalDetailRecord(primKey$, masterKey$);
clearMTRecord Syntax

clearMTRecord()

Description

Clears the fields of the current Message Tracking record. The database has to have been connected, via the openMTDBConnection command, previously. Please refer also to the topic Functions for Message Tracking Database, especially the command executeMTCommandWithMultiResult .

Parameter(s)

This command does not expect any parameters.

Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example: 
clearMTRecord();
closeMTDBConnection Syntax

closeMTDBConnection()

Description

This command is used to close a connection of a Message Tracking database (if it is still open due to the openMTDBConnection command). Refer also to the other Functions for Message Tracking Database.

Parameter(s)

This command does not expect any parameters.

Return Value

This command does not return a value.

Warning(s) /  Error(s)

If the closeMTDBConnection call returns an MTException the error

  • 2613

is created. The MTException is thrown, if the closing process fails, or the Message Tracking is working in an asynchronous mode.

Example 
closeMTDBConnection();
