addPrefixDataForChildMapping(data)

Description

Forces the next child mapping reader to read the provided data before the reading the input stream or file. Command has no effect if no child mapping is executed afterwards. For further information on child mappings, please refer to the topic Usage of Child Mappings.

Parameter(s)

Parameter

Type

Description

data

$

The data to read.
Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
local data$;

// gets the data in the current record
data$ = saveRecordContent();

// next child mapping will read, evaluate and convert this record too
addPrefixDataForChildMapping(data$);
addSuffixDataForChildMapping Syntax

addSuffixDataForChildMapping(data)

Description

Forces the next child mapping reader to read the provided data after reading - eventually aborted through the stopReading command - the input stream or file. Command has no effect if no child mapping is executed afterwards. For further information on child mappings, please refer to the topic Usage of Child Mappings.

Parameter(s)

Parameter

Type

Description

data

$

The data to read.
Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
// creates an 'UNZ' record for the child mapping
addSuffixDataForChildMapping("UNZ+1+3’");
startChildMapping Syntax

startChildMapping(mppg)

startChildMapping(mppg, in)

startChildMapping(mppg, in , use Reader )

startChildMapping(mppg, in , useReader, out)

startChildMapping(mppg, in , useReader, out , useWriter)

Description

Executes the specified child mapping immediately. The main mapping is paused until the child mapping is finished. The child mapping will reuse the doc reader/writer and/or the input/output file or stream if they are not explicitly specified. Please refer also to Usage of Child Mappings.

Parameter(s)

Parameter

Type

Description

mppg

$

The child mapping name.

in

$

The input file or stream name. Within BIS 6 only "", "automatically" and "mainSourceStream" are supported. 

"automatically": gets the next input attachment as input. Please work with the command hasMoreSourceAttachments().

"mainSourceStream": gets the complete main attachment as input

"": Reuses the current input stream.

useReader

$

A flag with the possible values "off" or "on" (default) to create a new or reuse the existing doc reader.

out

$

The output file or stream name. Within BIS 6 only "automatically" and "" are supported.

"automatically": creates a new output attachment.

"": Reuses the current output stream.

useWriter

$

A flag with the possible values "off" or "on" (default) to create a new or reuse the existing doc writer.
Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
startChildMapping("X_bmecat_to_I_Webcat");
startChildMapping("X_bmecat_to_I_Webcat", "automatically", "on", "automatically", "on");
startChildMapping("X_bmecat_to_I_Webcat", "mainSourceStream", "on", "webcat.dta", "off");
setErrorMapping Syntax

setErrorMapping(ErrorMappingName)

Description

Sets an error mapping. The mapping is called, if an error occurs in the mapping. Please refer to the topic Error Handling and Error Mappings. We recommend setting the option in the Option tab of the mapping.

Note: The command SetErrorMapping(null) resets the function.

Parameter(s)

Parameter

Type

Description

ErrorMappingName

$

Name of the error mapping, which is called if an error occurred. The source structure of the mapping has to contain an error record.
Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
       setErrorMapping("ErrorOrdersMap");

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
stopReading();
stopReadingOnError Syntax

stopReadingOnError(flag)

Description

Specifies whether a mapping will stop the processing if an error occurs. If this option is set to off, the mapping will end in errors as usual, but the complete source message will be read and processed. The default setting is on.

We recommend setting the option in the global Option tab of the mapping. The handling of warnings, errors, and the dependencies to other configurations is described in the topicError Handling.

Note: If the function is called multiple times in the mapping code, the last call will overwrite the earlier calls.

Note: If stopReadingOnError(false) is set the command isErrorOccurred() always returns false! Please use the command getLastErrorId() != -1 as workaorund.

Parameter(s)

Parameter

Type

Description

flag

$

A flag with the possible values "off" or "on" to activate or deactivate the behavior.
Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
stopReadingOnError ("on");
stopReadingOnError ("off");
