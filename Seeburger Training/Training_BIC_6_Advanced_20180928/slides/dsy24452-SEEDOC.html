XML and its rules

For more details please refer to

In order to arrange the plurality of possible XML documents and the associated number of different variations a set of rules is required. A set of rules provide clarity about the internal structure of an XML file. Thereby it becomes possible to respond to incoming scheduled XML files, for example in the area of EDI.
We distinguish two ways to structure XML files via a set of rules:

  1. Using Document Type Definitions
  2. Using Schemata

An XML document may be valid in several ways. That means, not only one version of an XML document is correct. Elements may be mandatory others optional.

For example the EDI use case: How can incoming files be processed without knowing which partner sends which structure? XML file is readable first when structure is known. Partners need to be imposed with rules as they may structure their XML documents.

Therefor it is important to know which elements and attributes occur in which structure (nested, tree structure). It is also important that, possible values are initialized with a limited selection (e.g. "f" or "m" for  gender, numeric or alphanumeric value, date only and so on).

These guidelines will ensure that XML files that correspond 100% to the structural specificatioin (DTD or schemata) sent by partner can be processed successfully.
