connectDB(name, connection, user, psw)

connectDB(name,connection,user,psw,driver)

connectDB(name,connection,user,psw,driver,transaction)

Description

Opens a connection to a database. The connection can be used by the mapDB and lookup commands. For further information refer to the topic Direct Database Access commands.

If  the connectDB command is executed more than once using the same name, the connection will be re-used, e.g.

connectDB("myConnection","jdbc:oracle:thin:@HOST:1521:DATABASE" ,"user"
,"psw","oracle.jdbc.driver.OracleDriver", "off");
connectDB("myConnection","jdbc:oracle:thin:@HOST:1521:DATABASE" ,"user"
,"psw","oracle.jdbc.driver.OracleDriver", "off");

Will create one connection only, which will remain open until the end of the conversion. If you use a different name, a new connection will be opened. BIC currently supports up to 10 different connections.
The connection will (per default) not use transactions; all statements executed using this connection will be immediately committed. If you want all statements to be committed, or rolled back depending on the result of the conversion, you have two options:

  • Execute all SQL statements within the same transaction context as the converter (only for database mappings).
  • Open the connection using the following syntax:
    connectDB("myConnection","jdbc:oracle:thin:@HOST:1521:DATABASE" ,"user"
,"psw","oracle.jdbc.driver.OracleDriver", "on");

    The on parameter will open a transaction that will be committed or rolled back at the end of the conversion.

Executing the Lookup

The following commands are required for executing the lookup and reading the result:

  • executeLookupDB(<lookupname>,<value 1>,<value 2>, <value 3>,<value 4>.)
  • getLookupFieldDB(<lookupname>,<field to retrieve>,<default value>)

The first function executes the SQL command.

Example:

openLookupDB("lookup1","myConnection","Partners","Name,address","Id=?");
executeLookupDB("lookup1",variable$,"","","");

Here variable$ contains the value for the parameter. The question mark is a place holder.

Use BIC numeric variables (variable ending with %) only if the field in the database is also numeric.

As shown above, the method can have up to 4 parameters. If further parameters have to be used, arrays can be applied, e. g.:

local params$[8];
m_stdDBmap.executeLookup("lookup1",params$);

The second function can be called as often as desired to retrieve one or more fields.

copy getLookupFieldDB("lookup1","Name","") to resultvariable$;

Some drivers (ODBC for SQL Server) require that the fields are read in the same order as they are defined in openLookup() command. Otherwise, the following error message will be displayed: [Microsoft][ODBC SQL Server Driver]Invalid Descriptor Index.

You can also use the existLookupDBRecord() command for checking whether the record was not found:

Dim found$;
...
executeLookupDB("lookup1",variable$,"","","");
if existLookupDBRecord("lookup_1")=1
	copy "1" to found$;
else
	copy "0" to found$;
endif

Note: BIC will generate a level 5 warning if the record is not found.

Or, you can also check if the field value was NULL using existLookupDBField() :

Dim exists$;
...
executeLookupDB("lookup1",variable$,"","","");
if existLookupDBField("lookup1","Fieldname")=1
	copy "1" to exists$;
else
	copy "0" to exists$;
endif

reuseSourceDBConnection

Attention: this command is deprecated. Please use the connection name SQLDocReader in the database access commands: openMapDB, openLookupDB.

Syntax

reuseSourceDBConnection()

Description

Reuses the connection to the source database for lookup commands. For further information please refer to the topic Direct Database Access commands.

Note: The connectDB () command is not required. The existing connection is called SQLDocHandler.

Attention: All the connections declared before the invocation of reuseSourceDBConnection() are overwritten by this command. They are not longer available for commands like getLookupDBField or mapDB. This behavior does not cause pending connections. All connections are closed after the conversion.

Parameter(s)

This command does not expect any parameters.

Return Value

This command does not return a value.

Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example 
reuseSourceDBConnection();
openLookupDB("MyLookup", "SQLDocHandler", "DESTTAB1","FIELD","cID='1'");
executeLookupDB("MyLookup");
