Child mappings

  • depending on an incoming file different child mappings can be called

  • e.g.: mixed incoming messages – 830 and 862

    • one child mapping for 830

    • one child mapping for 862

    • not required to create combined message structures to handle the files

  • different destination structure for one incoming flow

    • one mapping with INHOUSE as destination

    • one mapping with DB as destination

    • e.g.: incoming invoice or credit note will be mapped to INHOUSE and readable text file
