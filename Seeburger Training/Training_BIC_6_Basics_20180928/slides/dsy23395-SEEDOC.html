INHOUSE - Tutorial 1

Instruction

Illlustration

  1. Start BIC Mapping Designer and create a new project named InhouseAdapter.

  2. Import message structure msg_PURCHASE_ ORDER.xml

Check chapter "How to Operate BIC Mapping Designer" if you need further assistance.

  1. Open the new imported structure and extend with POS and POSTEXT

Open message structure

  1. Create a new Record POS below HEADER. Rightclick on HEADER and select Record, As subnode.

Create new Record

  1. Define the record details

  • Name = POS
  • Mandatory = yes
  • Min Occ = 1
  • Max Occ = 10

Record Details

  1. Create a new field RECORD below POS. Rightclick on POS and select Field, as subnode.

Create new field

  1. Define key field
  • Name = RECORD
  • Mandatory = yes
  • Min Occ = 1
  • Max Occ = 1
  • Minimal lenght = 3
  • Maximal length = 3
  • Type = KEY
  • Default value = POS

Define key details

  1. Define field details

  • Name = POSNUM
  • Mandantory = yes
  • Min Occ = 1
  • Max Occ = 1
  • Minimal lenght = 3
  • Maximal length = 3
  • Type = AN
  • Default value = "   "

Define field details

  1. Repeate the steps 4 to 8 according to the table in the exercise. 

  1. To doublecheck your result use the msg_PURCHASE_ ORDER_solution.xml in the classmaterial.

  1. You can use the function compare message descriptions to compare the solution with your manually created message structure.

  2. Right-click on the message structure and select Compare with.

start Compare message structure

  1. Select the compare data which is stored locally.

select compare data

  1. After clicking on ok you either see a popup with the information Compare completed. Files are identical.

  2. Or if they are not identical you will see a logfile with differences. By clicking on Show Tree you will see the differences in a graphical way.
