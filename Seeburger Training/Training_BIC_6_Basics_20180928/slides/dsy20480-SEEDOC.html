BIC MD GUI - Workspaces

  • Projects vs. workspaces:

    • Projects allow to organize/group mappings. Global mapping designer settings like defined datasources are valid for all projects listed in the project explorer.

    • Workspaces allow an organization on a higher level: Different mapping projects can be separated from each other having their own global mapping designer settings.

  • Typical scenario: Similar mapping projects from different BIS installations (e.g. QA and PRD environment). Advantages of using workspaces:

    • Clear separation. Reduces the risk of modifying the wrong mapping accidentally.

    • Define specific settings for each environment (data sources, default import/export paths)

  • A copy function allows to create a workspace based on an existing one to keep all settings.
