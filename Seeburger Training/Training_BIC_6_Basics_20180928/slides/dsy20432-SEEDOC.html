matches(string, regExp)

Description

Gets whether a string matches a certain regular expression. The Java regular expression matcher is used as the backbone for the related functionality. Please refer also to other string commands.

Parameter(s)

Parameter

Type

Description

string

$

The string.

regExp

$

The regular expression.
Return Value

Type

Description

%

A numeric value 0 or 1. (1 represents true).
Warning(s) / Error(s)

This command does not raise any warnings or errors.

Example

Introduction of regular expressions is beyond the scope of this documentation. Afterwards, only some basic information may be provided for a novice use of regular expressions in mappings.

Masking

Characters like ], }, \, ? etc. are a part of the regular expression syntax. In order to interpret them as a plain text they must be masked through a preceding \. (Results for the \ itself into a \\.)

Character Classes

Characters which should be matched through a regular expression may be enumerated, expressed as a range etc., e. g.

  • [abc] - a, b, or c
  • [^abc] - any character except a, b, or c (negation)
  • [a-zA-Z] - a until z or A until Z (inclusive).
Predefined Character Classes

For convenience a whole set of expected characters may be expressed as character classes, e. g.

  • . - any character
  • \d - a digit:[0-9]
  • \D - a non-digit: [^0-9]
  • \s - a white space character: [ \t\n\x0B\f\r]
  • \S - a non white space character: [^\s]
  • \w - a word character: [a-zA-Z_0-9]
  • \W - a non-word character: [^\w].
Quantifiers

Quantifiers are used to express the possible amount of occurrences, e. g.

  • ? - once or not at all
  • * - zero or more times
  • + - one or more times
  • {n} - exactly n times
  • {n,} - at least n times
  • {n,m} - at least n but not more than m times. 
local value$;
local anyChar$;
local digits$;
local matches%;

value$   = "abcdef";
anyChar$ = ".*";
digit$   = "\d*";

// assigns '1' - "abcdef" matches a sequence of any character
matches% = matches(value$, anyChar$);
 
// assigns '0' - "abcdef" is not a sequence of digits
matches% = matches(value$, digits$)
