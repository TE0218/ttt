How to use a MapFile - mapFile

Performs a value mapping operation on a previously open map file channel (see) . The command returns a value which is alphanumeric and which is the destination value for the mapping. The value is right trimmed before being returned (trailing spaces and tabs are removed).

There are two types:

  • mapFile(ChannelName, SourceValue, DefaultValue);

  • mapFile(ChannelName, SourceValue);

// Use Mapfiles
copy mapfile("Country",Country_Code$)    to Country_Name$;
copy mapfile("Currency",Country_Code$)  to Country_Currency$;
copy mapfile("Continent",Country_Code$) to Country_Continent$;
For more details please refer to openMapDB Syntax

mapFile

Attention: This command is deprecated. The should be used with database tables.

Syntax

mapFile(name, sourceValue);

mapFile(name, sourceValue, defaultValue)

Description

Performs a value mapping operation on a previously open map file channel (please refer to the command openMapFile). The command returns an alphanumeric value that is the destination value for the mapping. The value is right trimmed before being returned (trailing spaces and tabs are removed).

Mapfiles have to be stored in the directory ..\user\bicmd\mapfiles\  if you use the standard workspace up to version 6.3.3. If you use a different workspace the mapfiles have to be stored in the directory <workplace>\user\bicmd\mapfiles. When using version 6.3.4, you have to copy the mapfiles to the directory..\<workspace>\src\mapfiles\...

For further information please refer to the topic Mapfile and MapDB.

The command is deprecated. Please refer to the topic Migration from MapFile to MapDB .

Parameter(s)

Parameter

Type

Description

name

$

Name of the channel that is used. It has to be opened with the function.

sourceValue

$

This is a value expected by the function in the source column to produce a mapping. The destination value is the value residing in the same row, but in the destination column, defined by the  function.

defaultValue

$

Initial value if the source value is not found in the source column.
Return Value

Type

Description

$

The value of the destination column, if the source value is found in the source column. If the value is not found the default value or "" is returned.

Example 
! This map file maps two-letter country 
! codes to currency codes and geographical locations.
! country code | country name  | currency  | continent

DE,Germany,DM,Europe
US,United States of Americe,USD,North America
RUS,Russia,Ruble,Europe and Asia
BG,Bulgaria,Lev,Europe

To create a mapping that translates the country code (e.g., US) into currency (e.g., USD) write the following BIC BASIC code:

// We read a source value from the source message, 
// and before writing it to destination we do a mapping

// Open a map file channel for our file
openMapFile("Channel_1",         // any name would do
            "country_data.dta",  // name of the map file
            1,                   // 1 IS THE COLUMN FOR COUNTRY        
                                 // codes
            3,                   // 3 is the column for country        
                                 // currency symbols
            ",",                 // "," is the file column separator
            "country code not found!!!");  // default return value

dim CountryCode$;
dim CurrencyCode$;

copy SOURCE_MESSAGE:SOME_FIELD to CountryCode$;
CurrencyCode$ = mapFile("Channel_1", CountryCode$);
Copy CurrencyCode$ to DESTINATION_MESSAGE:WHATEVER_FIELD;
// End of mapping
