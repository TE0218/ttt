INHOUSE - Exercise 2

Import of the destination structure

  • Import the message description “order” from the file msg_Order.xml into your current project.  It would act as destination structure for the INHOUSE to INHOUSE-mapping.

Create a new mapping

  • New objects can be created on the project level by means of the context menu. In this way create a new mapping. Select as an input structure PURCHASE _ORDER from exercise before and as destination structure the before imported Order structure. Use the attached testfile to test your mapping.

Simple assignments

  • Search in both message structures comparable fields and assign the source fields to the corresponding target field. Simple assignments can be made via Drag&Drop.

  • If the destination field is smaller than the source field, check out the string manipulation commands.

  • It is important to test consistently the new created mapping. Take the relevant configuration of the test environment and test regularly the mapping. 

Tip: Comment your work steps.

Format date fields

  • You will find a date field in both used message structures. You have to do a format change (from CCYYMMDD to DDMMCCYY) while mapping the source field to the destination field. Use the format command.

Use of traces

  • While you were testing the mapping you have also controlled the log file after each run. Now you want to enrich the log file with additional information.

    • Record the order party. 

    • Further on you would like to understand by means of the log file, how many positions occurred in your application. Count them and then write them into the log file.
