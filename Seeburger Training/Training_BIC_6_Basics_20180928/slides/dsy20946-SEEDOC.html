Error Handling

There are two kinds:

  • Errors interrupt the conversion

  • Warnings make an entry to the BIC logfile but do not stop the conversion

The classification of an encountered problem is controlled by the settings in the configuration file (<BISRoot>\conf\bic\ini\juprefs.properties):

  • Error level: All warnings with errorlevel <= errorlevel get handled as an error

  • Warning level: All warnings with errorlevel <= warninglevel get handled as a warning

It is also possible to change the level of specific error codes. Example: There is a very strict handling required in general but values may be allowed to be truncated if the destination field is too short.
