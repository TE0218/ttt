Create an INHOUSE message structure

  • An INHOUSE message normally consist of the following attributes:

    • Records

    • Fields

    • CRLFs

  • After setting up a new message structure you can use the context menu to add the required records and fields to the message structure.

  • For INHOUSE message structures there are 2 special field types.

    • Key field - for identification

    • CRLF - create a new line at the end of a record.

For each record and field you have the chance to set the related details.

Error records will be handled in the Training BIC 6 Advanced.

For more details please refer to Record Properties

Field Properties

If a field is selected within the message tree, the properties of the field can be inspected via the Property panel, or in the corresponding table such as (excerpt):

General

Property

Description

Name

The name of the field.

Title

A short descriptive field title.

Description

A long description of the field.

Repetition

The repetition of the field name on the same level (parallel) - calculated automatically.

Position

Not in XML, SQL

The position of the field within the (parent) record - usually just a simple indexing of the field records.

In fixed length formats - such as INHOUSE, HWSW, EIAJ - the start position of the field in the record line.
Data

Occurrence

Property

Description

Mandatory

Determines whether the field is mandatory or optional. E. g. if it is optional with a min. occurrence of 3, it may not occur at all, but if it occurs, it has to occur at least 3 times. However, being mandatory with a min. occurrence of 3 means it has to occur always at least 3 times.

Min. Occ.

The minimal occurrence of the field. Not in SQL.

Max. Occ.

The maximal occurrence of the record. Not in SQL.

Length

Property

Description

Min. and max. length

The the minimal or maximal length of the field depending on the field type. If the field type has a fixed length, min. and max. length have to be equal. If the field type has a variable length, the min. length has to be < then the max. length. If the field is shorter then the min. length, an error will occur during the conversion.

Range

Property

Description

Range (from and to)

A numerical range which limits the a numerical value.

Incl. lowest/highest range value

Determines whether a numerical value may be equal to one or both range boundaries.

Not in XML, SQL.

Classifies the field to be a qualifier or not. This is used further to control the command during a compare operation.

Binary ID

Only in INHOUSE, HWSW, EIAJ - See here, the section for the Binary ID.

Field value encoding

Not in SQL. The field value may be encoded.

  • Char - plain text, encoded as specified through the mapping encoding.
  • Base64 - base 64 encoded.
  • Hexbinary - hexbinary encoded.

Field value treatment

Not in SQL.

  • Value - the content of a field is enterpreted just as a plain value.
  • Attachment - the content of a field is enterpreted as an attachment ID. If a mapping is performed the content which is associated with the attachment ID will be mapped.
  • Both - If a mapping is performed at first the simple value is mapped and the attachment content afterwards.

Default Value

The default value of the field - only filled, if the field has a fixed value and the field is not filled during the mapping process.

Compare

Determines whether and how the record should participate in compare operations. See Compare Mode.

Format

The format of the field, please refer also to the topic Format strings.

Enumeration

A list of values regarded to be allowed field values. If the syntax check level is set to 3, the field value is checked against this list.

The type of the belonging table key field column.
