Writer

There are two different types of writing behavior:

  • EDI formats, XML

    • Writes the data at the end of a mapping

    • All values of the destination message can be read and modified during runtime before the actual message object is really created

    • To reduce memory consumption there is a header/trailer handling for EDI messages which saves finished messages/documents but includes a special handling for service records

  • SQL

    • The behaviour depends on the configuration of the writer (e.g. transaction mode = on/off)

    • Written data cannot be read
