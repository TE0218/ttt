BIC MD GUI - Compare Messages, Mappings, Procedures

  • The Compare can be used to compare Message structures, Mappings and Procedures.

  • Start compare with a right-click on an object or via Menu → Build → Compare. Afterwards the dialog below will open.

A common use case is to compare a structure located in repository with a structure stored locally on the file system.

For more details please refer to Compare

Messages, mappings, and global procedures may be compared with each other. Backbone for the ability is the BIC Converter which supports comparing of converted data with reference data as it can be found in the test environment. Here the messages, mappings, and global procedures are just interpreted as XML files which could have been involved in a conversion. The belonging message formats for compare are fix and can be found after each installation in [Installation Path]\conf as msg_CompareGlobalProcedures.xml, msg_CompareMapping.xml and msg_CompareMessage.xml.

Invocation
  • Via context menu called if the selection is on a node which may be compared - entry Compare with.
  • Through dragging a proper node on another node to compare both with each other.

The compare dialog appears in both cases.

Compare Dialog

Screenshot 0-1: Compare Dialog

If not filled yet completely, the user has to enter the missing data. If the data is somehow erroneous a continuation - e.g. clicking on the OK button - is impossible.

The colored background marks the text field which is currently receiving the user input. The point of interest is here, that the user can click on any node in the background project view, and the focused text field will be filled with the corresponding name.

For a greater flexibility even files from the physical file system can be chosen through a click on the Browse button next to each text field. Choosing a file from the physical file system has to assure that only equal objects are compared with each other, i. e. messages should only be compared with messages, mappings with mappings and global procedures with global procedures.

If you click on the OK button, the compare operation continues as described for the compare feature included in the test environment.

Typical Use Cases Repository

A favorite usage of the compare tool is to detect the differences of the local source code with the source code in the repository. Therefore the objects in the repository are imported into another (new) project in BIC Mapping Designer. Comparing now project-wide unveils the differences which a user created in between through editing his local source code in BIC Mapping Designer compared to his previously committed code in the repository, or compared to the contributions into the repository of another code developer.
